─────────────────────────────────────────────────────
🔴 PRIORITAS 1 — Enrollment System (HARUS DIKERJAKAN)
─────────────────────────────────────────────────────
  Mengapa ini prioritas tertinggi?
  → Enrollment adalah INTI dari LMS. Tanpa ini, user tidak bisa
    "mendaftar" ke kursus sama sekali. Halaman detail sudah ada
    tombol "Enroll", tapi belum terhubung ke backend.
  → Invoice & Transaction schema SUDAH ADA — tinggal buat logic-nya.
  → Ini BLOCKING semua fitur lain: progress tracking, sertifikat,
    dashboard dinamis, semuanya bergantung pada enrollment.

  Yang perlu dikerjakan:
  → POST /api/enrollments — daftarkan user ke kursus
  → GET  /api/enrollments/me — list kursus yang diikuti user
  → Update tombol Enroll di course-detail.tsx
  → Tampilkan status "Sudah Enroll / Belum" di course card

─────────────────────────────────────────────────────
🔴 PRIORITAS 2 — Dashboard Dinamis (BLOCKING UX)
─────────────────────────────────────────────────────
  Mengapa prioritas tinggi?
  → Dashboard adalah halaman PERTAMA yang dilihat user setelah login.
    Saat ini data chart & kursus masih hardcoded — tidak mencerminkan
    aktivitas user yang sebenarnya.
  → Setelah enrollment selesai, data nyata sudah tersedia — tinggal
    ditampilkan di dashboard.
  → User experience > fitur baru. Dashboard yang dinamis akan langsung
    dirasakan oleh semua pengguna.

  Yang perlu dikerjakan:
  → Query enrollment nyata user dari DB di dashboard/page.tsx
  → Ganti list kursus hardcoded dengan kursus yang di-enroll
  → Progress tracking per lesson (LessonProgress model sudah ada)
  → Todo list persisten ke database

─────────────────────────────────────────────────────
🟡 PRIORITAS 3 — Video Player & Progress Tracking
─────────────────────────────────────────────────────
  Mengapa prioritas sedang?
  → Ini adalah fitur core LMS kedua setelah enrollment.
    Tanpa video, LMS tidak beda dengan website statis.
  → LessonProgress model sudah ada di schema — tracking sudah didesain.
  → Bisa mulai dengan YouTube embed (gratis) sebelum pindah ke Cloudinary.

  Yang perlu dikerjakan:
  → app/courses/[slug]/learn/page.tsx — halaman belajar
  → Embed YouTube / Cloudinary video player
  → POST /api/progress — update progress saat lesson selesai
  → UI sidebar silabus dengan highlight lesson yang sudah selesai

─────────────────────────────────────────────────────
🟡 PRIORITAS 4 — Sistem Pembayaran (Midtrans)
─────────────────────────────────────────────────────
  Mengapa bisa belakangan?
  → Invoice & Transaction schema sudah ada (dari standard tim).
  → Kursus gratis bisa di-enroll tanpa payment — prioritas 1 & 2
    bisa selesai dulu sambil payment dikerjakan paralel.
  → Midtrans memerlukan akun merchant & environment terpisah.

  Yang perlu dikerjakan:
  → Integrasi Midtrans Snap API
  → POST /api/invoices — buat invoice sebelum payment
  → Webhook handler untuk konfirmasi pembayaran
  → Halaman order history
